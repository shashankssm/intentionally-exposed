# AWS IAM User Provisioning with Terraform

This Terraform project creates multiple IAM users, enables AWS Management Console access, creates an IAM group, and adds all users to the group.

## Resources Created

- IAM Group
  - `Platform-Developers`

- IAM Users
  - Tom
  - Bob
  - John

- Console Login
  - Management Console access enabled
  - Auto-generated secure password
  - Password reset required on first login

- Group Membership
  - All users are added to the `Platform-Developers` IAM group.

## Configure AWS Credentials

Create a `.env` file in the project root.

```env
AWS_ACCESS_KEY_ID=YOUR_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY=YOUR_SECRET_ACCESS_KEY
AWS_REGION=ap-south-1
```

> **Important**
>
> Never commit the `.env` file to Git.

## View Generated Passwords

Terraform generates secure passwords for each user.

To view them:

```bash
terraform output generated_passwords
```

For JSON output:

```bash
terraform output -json generated_passwords
```

> Store these passwords securely and share them with the respective users.

## Security Best Practices

- Never commit AWS credentials.
- Never commit the `.env` file.
- Rotate AWS access keys periodically.
- Enable MFA for all IAM users.
- Follow the Principle of Least Privilege when attaching IAM policies.
- Store Terraform state securely (e.g., Amazon S3 with state locking via DynamoDB) for production deployments.

## Notes

- IAM users are created using Terraform's `for_each` construct.
- Passwords are randomly generated by AWS during login profile creation.
- Password reset is enforced on first sign-in.
- AWS credentials are loaded from environment variables defined in the `.env` file, keeping secrets out of the Terraform configuration.
